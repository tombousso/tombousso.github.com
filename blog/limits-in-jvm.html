Limits in the JVM

05 November 2017

Scattered throughout chapter 4 of the JVM spec are the various structs which make up a classfile. In these structs are entries like u2 fields_count and u2 methods_count, which limit both the number of fields declared in a class and the number of methods declared in a class to 65535. Code attributes have u4 code_length, and so you might expect that a method can have 2^32-1 bytecodes, but the limit is actually 65535:

The value of code_length must be greater than zero (as the code array must not be empty) and less than 65536.
— https://docs.oracle.com/javase/specs/jvms/se9/html/jvms-4.html#jvms-4.7.3

While humans probably won’t surpass these limits while writing Java code, classfiles are often generated by automated tools or from languages other than Java, and 64K of bytecodes is not a huge amount. These limits should be increased (to either 32-bit, 64-bit, or varint), however increasing them requires a significant number of changes to the JVM spec.
